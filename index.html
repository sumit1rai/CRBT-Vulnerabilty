1. Ubuntu System-Wide Package Updates (Apply to ALL 5 Servers)

Packages
⚡ This single command addresses the bulk of Sev 4–5 Ubuntu package CVEs: OpenSSL, GnuTLS, nginx, pip, rsync, Lodash, redis, libarchive, inetutils, BIND, LibTIFF, OpenSSH, pyOpenSSL, PyJWT, urllib3, Memcached, wheel, kmod, libpng, jq, shell-quote, pyasn1, Twisted, Composer, libssh/libssh2, nghttp2, dpkg, NASM, Bouncy Castle, Symfony, Vim, GDK-PixBuf and many more.
1
Connect to each server and run a full system update
sudo apt-get update
sudo apt-get upgrade -y
sudo apt-get dist-upgrade -y
sudo apt-get autoremove -y
Run this on all five servers: .6, .7, .8, .10, .11, .12 (one by one or via Ansible/parallel-ssh).
2
Reboot each server (REQUIRED for kernel and AppArmor patches)
sudo reboot
Kernel patches only take effect after reboot. Schedule maintenance windows for production servers.
3
Verify patched kernel and key packages
uname -r                        # verify new kernel version
openssl version                 # check OpenSSL version
dpkg -l | grep -E 'nginx|redis|openssh-server|bind9|rsync'
ℹ️ For database servers (.11, .12) — stagger reboots to maintain DB HA. Reboot replica first, promote if needed, then reboot primary.

2. Replace End-of-Life (EOL) Software

EOL
⚠️ EOL software receives NO security patches. These MUST be replaced — apt-upgrade alone will NOT fix these.
A
Apache Struts 1 → Migrate to Struts 2.5.x / 6.x (or Spring MVC)
Struts 1 has been EOL since 2013 and has no patches. Applications using it must be migrated.
# Identify apps using Struts 1 (look for struts-1.x.jar)
find / -name "struts-1.*.jar" 2>/dev/null
find /opt /var/lib -name "struts.jar" 2>/dev/null

# Migration path:
# → Struts 2: https://struts.apache.org/migration.html
# → For new apps, prefer Spring Boot / Jakarta EE
B
Apache Tomcat 8.5.x → Upgrade to Tomcat 10.1.x or 11.0.x
Tomcat 8.5 is EOL. Upgrade to 10.1.x (Jakarta EE 10) or 11.0.x (latest LTS).
# Check current Tomcat version
/opt/tomcat/bin/version.sh
catalina.sh version

# Download latest stable
wget https://dlcdn.apache.org/tomcat/tomcat-10/v10.1.x/bin/apache-tomcat-10.1.x.tar.gz

# Stop old Tomcat, backup webapps, install new version
sudo systemctl stop tomcat
sudo tar -xzf apache-tomcat-10.1.x.tar.gz -C /opt/
sudo cp -r /opt/tomcat/webapps/* /opt/apache-tomcat-10.1.x/webapps/
sudo systemctl start tomcat-10
⚠️ Tomcat 10+ uses Jakarta EE namespaces (javax.* → jakarta.*). Test apps carefully before migrating to production.
C
Apache Log4j 1.x → Migrate to Log4j 2.24+ or SLF4J/Logback
Log4j 1.x is EOL since 2015 and has critical RCE vulnerabilities.
# Find all log4j 1.x jars in the system
find / -name "log4j-1.*.jar" -o -name "log4j.jar" 2>/dev/null | grep -v proc

# In Maven projects, replace:
# 
#   log4j
#   log4j   ← REMOVE THIS
# 

# Add Log4j 2:
# 
#   org.apache.logging.log4j
#   log4j-core
#   2.24.3
#
D
Node.js 18.x (EOL) → Upgrade to Node.js 22.x LTS
Node.js 18 reached EOL in April 2025. Upgrade to Node.js 22 (current LTS).
# Using nvm (recommended)
nvm install 22
nvm use 22
nvm alias default 22

# Or using NodeSource apt repo
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verify
node --version   # Should show v22.x.x
npm --version

# Update npm itself
npm install -g npm@latest

# Also update lodash to latest in your projects:
npm install lodash@latest  # or: npm audit fix

3. Linux Kernel — Local Privilege Escalation (Dirty Frag / LPE)

Kernel
⚠️ CVE-2026-43284, CVE-2026-43500 (Dirty Frag), CVE-2026-46300 (Fragnesia), USN-8254-1, USN-8278-1, USN-8179-1 all require kernel upgrade + reboot.
1
Update kernel via apt (Ubuntu 24.04)
sudo apt-get update
sudo apt-get install --only-upgrade linux-image-$(uname -r) linux-headers-$(uname -r)
# Or upgrade all kernel packages:
sudo apt-get install -y linux-generic linux-headers-generic linux-image-generic
2
Reboot and verify
sudo reboot
# After reboot:
uname -r   # Confirm new kernel version
apt-cache policy linux-image-generic   # Confirm installed == candidate
3
Interim mitigation (if reboot cannot happen immediately)
Restrict local user access to minimize LPE risk until reboot is possible:
# Disable user namespaces (mitigates some LPE vectors)
sudo sysctl -w kernel.unprivileged_userns_clone=0
echo 'kernel.unprivileged_userns_clone=0' | sudo tee -a /etc/sysctl.conf

# Ensure non-root users cannot run dangerous SUID binaries
sudo chmod u-s /usr/bin/newgrp /usr/bin/pkexec 2>/dev/null || true
ℹ️ sysctl mitigation is temporary. A full reboot with updated kernel is the only true fix.

6. Apache Tomcat 9.0 / 11.0 — Padding Oracle, Directory Traversal, DoS

App
1
Upgrade Tomcat 11.0 to 11.0.9+ and Tomcat 9.0 to 9.0.102+
# Check version
/opt/tomcat/bin/version.sh

# Download latest Tomcat 11.0 from:
# https://tomcat.apache.org/download-11.cgi

# Download latest Tomcat 9.0 from:
# https://tomcat.apache.org/download-90.cgi

sudo systemctl stop tomcat
# backup existing install, then replace with new version
sudo tar -xzf apache-tomcat-11.0.x.tar.gz -C /opt/

2
Harden EncryptInterceptor (CVE-2026-29146 mitigation)
# In server.xml, ensure EncryptInterceptor uses strong settings:
# <Receiver className="org.apache.catalina.tribes.transport.nio.NioReceiver"
#   encryptionAlgorithm="AES/GCM/NoPadding"
#   encryptionKeyLength="256" />
3
Mitigate HTTP/2 DoS — limit concurrent streams
# In server.xml connector:
<Connector port="8443" protocol="org.apache.coyote.http11.Http11NioProtocol"
    maxConcurrentStreamExecution="20"
    maxHeaderCount="100" />
4
Fix CVE-2026-43512 — Digest Auth unknown user bypass
Ensure you run Tomcat 9.0.102+ which patches this. If you use Digest auth, switch to client certificates or JWT-based auth as a stronger alternative.

Post-Remediation Verification Checklist

  • Run sudo apt-get update && sudo apt-get upgrade -y on all 5 servers and confirm 0 packages pending
  • Reboot all servers and verify new kernel via uname -r
  • Confirm AppArmor enforce mode via aa-status
  • Verify Tomcat version upgraded to 11.0.9+ / 9.0.102+ via catalina.sh version
  • Confirm Struts 1 jars are fully removed from all application deployments
  • Confirm Log4j 1.x jars are fully removed and applications use Log4j 2.24+ or Logback
  • Verify Node.js 22.x is active: node --version
  • Verify Tomcat 8.5 is no longer installed or running
  • Run Qualys re-scan to confirm resolved vulnerabilities
  • Review Unix accounts with root UID/GID (reported in scan) and remove unauthorized accounts
  • Restrict open TCP listening ports to only necessary services
  • Enable unattended-upgrades for ongoing security patches
# Enable automatic security updates (recommended)
sudo apt-get install unattended-upgrades
sudo dpkg-reconfigure --priority=low unattended-upgrades

Quick-Start: Run on All 5 Servers (via Ansible or parallel-ssh)

If you use Ansible or parallel-ssh to manage these servers, here is a combined shell script to apply all apt-based patches at once. EOL software and application upgrades still require manual steps per app.
#!/bin/bash
# CRBT Nodes — Ubuntu Patch Script
# Run as: sudo bash crbt_patch.sh
# Apply to: 172.18.229.6, .7, .8, .10, .11, .12

set -e
echo "=== Updating package lists ==="
apt-get update -q

echo "=== Upgrading all packages ==="
DEBIAN_FRONTEND=noninteractive apt-get upgrade -y -q
DEBIAN_FRONTEND=noninteractive apt-get dist-upgrade -y -q

echo "=== Installing security hardening ==="
apt-get install -y unattended-upgrades apparmor apparmor-utils

echo "=== Reloading AppArmor ==="
systemctl restart apparmor
aa-enforce /etc/apparmor.d/* 2>/dev/null || true

echo "=== Cleaning up ==="
apt-get autoremove -y -q
apt-get autoclean -q

echo "=== Patch complete. REBOOT REQUIRED ==="
echo "Run: sudo reboot"
⚡ Ansible playbook approach: use ansible-playbook -i inventory.ini patch_crbt.yml with the shell module wrapping the above script for all 5 hosts.
